# Contributing

This is a **template repository** — its primary use is to be copied (via *“Use this template”*, clone, or ZIP) and filled with your own world. You don't need to contribute back to use it.

That said, improvements to the *scaffold itself* are welcome:

## Good contributions

- Clearer rules or wording in `CLAUDE.md`.
- New domain MOCs or entity-template fields that generalize across many worlds.
- Better Dataview queries for the dashboards in `HOME.md` (must work on free Dataview).
- Fixes to the player-facing HTML/CSS publishing layer in `notes/canon/`.
- Documentation, examples, and onboarding improvements.

## Ground rules

1. **No world data.** This repo is generic by design. Never commit real campaign content, character names, source-document paths, or personal data. The single worked example (`entities/_EXAMPLE_aldric-thorne.md`) is invented filler and clearly marked.
2. **No paid or private tools required.** Everything must work with free Obsidian + the free Dataview plugin. Paid options (e.g. Obsidian Publish) may be *mentioned as optional*, never required.
3. **Keep placeholders generic.** Use `{{DOUBLE_BRACE}}` placeholders for anything a user must fill in.
4. **Preserve the discipline.** The whole point is source-cited, conflict-flagged canon. Don't add patterns that encourage un-sourced "fact by vibes."

## How

Fork → branch → make your change → open a pull request describing what it improves and why it generalizes. Small, focused PRs are easiest to review.
